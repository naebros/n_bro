No git required — copy the Worker source and deploy it directly from the Cloudflare dashboard.

Make sure you have these before starting.

Node.js 18+ Cloudflare account Telegram bot token Groq API key

1
Install Wrangler CLI
bash

2
Get the source code

Project

Groq AI Bot

A Telegram bot running on a Cloudflare Worker. Supports text, voice, and image messages — powered entirely by Groq's free API.

JavaScript Cloudflare Workers Free to deploy Telegram

What it does

The bot runs as a Cloudflare Worker — no server needed. Telegram sends updates to your Worker via webhook, the Worker calls Groq's API, and sends the reply back. History is kept per user in memory for up to 10 turns.

Text chat
Full conversation history per user, trimmed automatically to the last 10 exchanges.
Voice messages
Transcribed with Groq Whisper (whisper-large-v3-turbo), then answered as regular text.
Photo analysis
Images analysed by meta-llama/llama-4-scout-17b-16e-instruct with optional caption prompt.
Commands
/start, /help, /clear — command messages are auto-deleted from the chat.

Prerequisites

Cloudflare account (free) Telegram bot token via @BotFather Groq API key — console.groq.com Wrangler CLI

Deploy to Cloudflare

1
Install Wrangler and log in
bash 
npm install -g wrangler
wrangler login
2
Get the source code
Open the file, copy the code, and paste it into the Cloudflare dashboard editor.

3
Set secrets in Cloudflare
Secrets are stored securely — never in code or wrangler.toml.
bash 
wrangler secret put TELEGRAM_TOKEN
wrangler secret put GROQ_API_KEY
wrangler secret put GROQ_MODEL
wrangler secret put GROQ_VISION_MODEL
4
Deploy the Worker
bash 
wrangler deploy
5
Register the webhook
Open this URL once in your browser — it registers the Telegram webhook automatically. Replace with your actual Worker URL.
url 
https://your-worker.workers.dev/setup
You should see {"ok":true} in the response. The bot is now live.

Environment variables

These are set as Cloudflare secrets (not in code). All four are required.

secrets 
TELEGRAM_TOKEN        # from @BotFather
GROQ_API_KEY          # from console.groq.com
GROQ_MODEL            # e.g. llama-3.3-70b-versatile
GROQ_VISION_MODEL     # e.g. meta-llama/llama-4-scout-17b-16e-instruct

Bot commands

Command messages are automatically deleted from the chat after processing.

/start
Clears session history and shows a welcome message with available commands.
/help
Lists all bot capabilities — text, voice, photo, and commands.
/clear
Shows an inline confirm/cancel button before wiping the user's conversation history.

Notes

Conversation history lives in the Worker's memory (userSessions Map) and is lost on Worker restart or cold start. For persistent history, you'd need Cloudflare KV or D1.

Component Kit

A reusable component library with buttons, inputs, modals, tooltips, and more — designed in Figma and exported to production-ready code.

Figma TypeScript Accessible

What's included

Buttons
Primary, secondary, ghost, icon-only. With icon-reveal hover animations.
Inputs
Text, password, search, textarea. With validation states and labels.
Modals
Animated dialogs with overlay, keyboard trap, and accessible roles.
Tags & Badges
Pill-shaped labels in multiple colour variants with icon support.

Usage

Import components directly. Each one is fully typed and uses CSS tokens for theming.

TypeScript 
import { Button, Modal, Input } from "@naebro/component-kit";

export default function App() {
  return <Button variant="primary">Click me</Button>;
}

Component Kit is still in active development. Some APIs may change between minor versions. Pin to a specific version in production.

AI Playground

An interactive sandbox for testing AI prompts, comparing model outputs side-by-side, and exploring creative generation workflows.

Next.js Streaming Vercel

Features

The Playground is built for speed and experimentation. Every feature is designed around fast iteration.

Side-by-side comparison
Run the same prompt against multiple models simultaneously and compare outputs.
Streaming responses
Tokens stream in real time — no waiting for the full response.
Save presets
Save, fork, and share your favourite prompt configurations.
Parameter control
Adjust temperature, max tokens, top-p, and more from the UI.

FAQ

Common questions about Naebro's projects.

Is everything free?

Yes. All projects are open source and free to use. Some projects (like Groq AI Bot) require free third-party API keys, but there are no paid tiers in the projects themselves.

Where's the source code?

Everything lives on GitHub at github.com/naebros. You can also download the full repo as a zip from the homepage.

Can I contribute?

Pull requests are welcome. Open an issue first to discuss what you'd like to change, especially for larger features.

How do I report a bug?

Open an issue on the relevant GitHub repository. Include steps to reproduce, expected behaviour, and actual behaviour. Screenshots or logs are very helpful.
